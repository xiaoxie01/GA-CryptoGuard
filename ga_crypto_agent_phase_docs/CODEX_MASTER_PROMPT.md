# Codex 执行总提示词

你正在实现 GA CryptoGuard，这是一个基于 GenericAgent 的币安合约市场研究 Agent。

## 项目边界

- 只做行情分析、飞书预警、模拟盘、机会监控、复盘、自进化。
- 不接实盘。
- 不调用真实交易接口。
- 不输出确定性收益承诺。
- 所有交易相关动作只写入模拟盘。
- 用户消息优先级必须高于后台定时任务。
- 后台任务不得污染用户飞书会话上下文。

## 执行方式

请按 `phases/phase_xx_*` 顺序逐阶段实现。每个阶段文件夹包含：

- `README.md`：阶段目标和范围。
- `IMPLEMENTATION.md`：具体开发任务。
- `DATA_MODEL.md`：数据库与配置变更。
- `API_AND_TOOLS.md`：需要新增或修改的工具函数。
- `ACCEPTANCE.md`：验收标准。
- `TEST_CASES.md`：测试用例。
- `REVIEW_CHECKLIST.md`：审查清单。

实现一个阶段时：

1. 先阅读 `global/` 下所有全局文档。
2. 阅读当前阶段全部文档。
3. 检查前置阶段是否已经完成。
4. 只实现当前阶段范围内的功能，不提前实现后续阶段复杂逻辑。
5. 为新增逻辑补充单元测试和集成测试。
6. 更新迁移脚本和配置样例。
7. 所有任务应幂等、可重试、可审计。
8. 所有时间统一使用 UTC。
9. 所有 K 线分析只能使用已收盘 K 线，禁止未来函数。
10. 所有 GA 输出必须结构化保存。

## 验收方式

每阶段完成后，必须能通过：

- 自动化测试。
- 手动飞书命令测试。
- 数据库记录检查。
- 日志检查。
- 任务重跑幂等性检查。
- 用户消息与后台任务隔离检查。

若验收失败，先修复当前阶段，不进入下一阶段。
