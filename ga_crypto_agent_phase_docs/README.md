# GA CryptoGuard 阶段化实施文档包

本包用于指导 Codex / 自动化编码助手按阶段实现一个“不接实盘”的币安合约市场研究 Agent。

系统定位：

> 不接实盘的合约市场研究 Agent + 飞书交互式预警系统 + 模拟盘验证系统 + 自进化策略实验室。

## 强制边界

- 不接实盘。
- 不调用 Binance 下单接口。
- 不保存具备交易权限或提现权限的 API Key。
- 行情数据可以使用公开接口。
- 所有交易动作仅进入模拟盘。
- 所有 GA 自进化动作必须经过候选版本与影子测试，不允许直接覆盖 active 策略。

## 阶段总览

| 阶段 | 名称 | 目标 |
|---|---|---|
| Phase 01 | 稳定性与可观测性 | 保证系统稳定运行、用户消息不被后台任务影响 |
| Phase 02 | 数据闭环完善 | 分析、信号、模拟盘、复盘都可追踪 |
| Phase 03 | 信号分级机制 | 降低飞书噪音，提高预警质量 |
| Phase 04 | 机会监控强化 | 支持“现在不做，但继续盯机会” |
| Phase 05 | 模拟盘执行质量 | 不只记录盈亏，还要记录交易计划质量 |
| Phase 06 | 价格行为模块 | 建立稳定的结构识别能力 |
| Phase 07 | 动能模块 | 判断启动、延续、过热、衰竭 |
| Phase 08 | 趋势阶段模块 | 判断初期、中期、末期、震荡、转折 |
| Phase 09 | SMC 与订单流模块 | 提升流动性和入场质量判断 |
| Phase 10 | 缠论模块 | 加入分型、笔、中枢、背驰、买卖点候选 |
| Phase 11 | 复盘归因 SOP | 每笔模拟单平仓后自动归因 |
| Phase 12 | 策略版本管理 | 所有策略变化可追踪、可回滚 |
| Phase 13 | 影子测试 | 候选策略先验证再升级 |
| Phase 14 | 历史回放 / 回测 | 利用长期 K 线快速积累样本 |
| Phase 15 | 自进化闭环 | 复盘 → 补丁 → 影子测试 → 升级 |

## 使用方式

1. 先读 `CODEX_MASTER_PROMPT.md`。
2. 读 `global/` 全局文档。
3. 按 `phase_manifest.yaml` 顺序进入各阶段文件夹。
4. 每阶段完成实现、测试、验收和审查后，再进入下一阶段。
