# Phase 09：SMC 与订单流模块

## 阶段目标

SMC 与订单流模块 是 GA CryptoGuard 在 MVP 后推进路线中的第 9 阶段。

本阶段只实现当前范围内的功能，不提前实现后续阶段高级能力。

## 前置条件

- 已阅读 `global/` 全局设计。
- 已执行 `global/schema.sql` 基础建表或对应迁移。
- 前置阶段已通过验收。
- 系统仍保持“不接实盘”的强制边界。

## 本阶段主要任务

- 识别 Liquidity Sweep
- 识别 FVG / OB
- Premium/Discount
- CVD 与主动买卖比
- 价格-CVD 背离

## 本阶段完成后应达到

- SMC 输出流动性事件
- 订单流能确认或否定入场
- 无订单流数据时可降级运行
- 高质量信号包含 SMC/订单流证据
