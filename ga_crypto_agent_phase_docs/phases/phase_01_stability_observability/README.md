# Phase 01：稳定性与可观测性

## 阶段目标

稳定性与可观测性 是 GA CryptoGuard 在 MVP 后推进路线中的第 1 阶段。

本阶段只实现当前范围内的功能，不提前实现后续阶段高级能力。

## 前置条件

- 已阅读 `global/` 全局设计。
- 已执行 `global/schema.sql` 基础建表或对应迁移。
- 前置阶段已通过验收。
- 系统仍保持“不接实盘”的强制边界。

## 本阶段主要任务

- 实现 /status 系统状态指令
- 完善 scheduler_runs / agent_jobs / task_locks
- 验证用户消息优先级高于后台任务
- 飞书推送去重
- 错误日志查询

## 本阶段完成后应达到

- 连续运行 72 小时无致命错误
- 后台 15m 任务运行时用户消息仍能优先响应
- 所有定时任务都有 scheduler_runs 记录
- 重复飞书事件不会重复处理
